1. Purpose

This SOP defines a standard process to identify, analyze, and resolve backup failures in the IT infrastructure. The goal is to ensure data integrity, regulatory compliance, and business continuity by minimizing the Mean Time To Resolve (MTTR) for backup incidents.

3. Responsibility Matrix

Role Responsibility
L1 Support Daily dashboard monitoring, verification, initial retry, ticket logging.
L2 Support Log analysis, space clearing, script debugging, manual execution.

4. Common Failure Types

Failure Type Example / Symptom
Space Issue Disk usage > 95%, "No space left on device" error.
Permission Issue "Access Denied", "Read-only file system", User password expired.
Service Issue SQL Writer Service stopped, Cron daemon stuck.
Network Issue NAS IP unreachable, VPN tunnel disconnected.
File Lock Database in use, "File is being used by another process".

  • Any "Skipped" backup is considered a Failure for audit purposes.
  • Manual retries must be documented in the Daily Operations Report.
