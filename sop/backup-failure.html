1. Purpose

To define the standard procedure for identifying, troubleshooting, and resolving backup failures in Linux & Windows environments to ensure zero data loss.

2. Scope

  • Linux Servers (RHEL, CentOS, Ubuntu)
  • Windows Servers (2016, 2019, 2022)
  • Database Backups (SQL/MySQL)

3. Responsibility Matrix

RoleResponsibility
L1 SupportDaily Verification & Ticket Logging
L2 SupportFailure Analysis & Manual Retry
SysAdminRoot Cause Analysis (RCA) & Fix

4. Failure Handling Flow

  1. Identify: Check dashboard/logs for error messages.
  2. Validate: Check disk space (`df -h`) and services.
  3. Action: Restart service, clear space, retry backup.
  4. Verify: Confirm new file creation and timestamp.
